### Why This Matters

From our research:
- **99% of programs**: Grades only

### Scoring Algorithm

```python
# Base score from tier weights
base_score = sum(tier_weights[activity.tier] for activity in activities)

# Bonus for diversity (3+ different categories)
diversity_bonus = 0.5 if len(unique_categories) >= 3 else 0

# Bonus for leadership (up to 2 points)
leadership_bonus = min(leadership_count * 0.5, 2)

# Total score (capped at 20)
total_score = min(base_score + diversity_bonus + leadership_bonus, 20)
```

```bash
Python 3.9-3.13
pip install -r requirements.txt
```

### Installation

```bash
# Clone repository
git clone https://github.com/yourusername/University_Admission_Prediction_Project.git
cd University_Admission_Prediction_Project

# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

### Testing

```bash
# Run all tests
pytest tests/ -v

# Run specific module tests
pytest tests/test_features/test_grade_parser.py -v
pytest tests/test_features/test_extracurricular_parser.py -v

# Check coverage
pytest --cov=src tests/
```

## Technical Deep Dive

### Design Patterns Used

This project employs several OOP design patterns:

1. **Abstract Base Class Pattern** - `BaseModel` and `BaseEncoder` define consistent interfaces
2. **Strategy Pattern** - Multiple encoding strategies are interchangeable
3. **Template Method Pattern** - Base classes orchestrate workflow
4. **Pipeline/Composition Pattern** - Complex transformations built from simple components

### Feature Engineering

Raw data is transformed through domain-specific encoders (GPA, University, Program, EC) into a normalized design matrix for model training.

## Actionable Insights

### For Software Developers

1. **Prevent Data Leakage** - DesignMatrixBuilder.fit() must ONLY be called on training data
2. **Numerical Stability** - Sigmoid function needs special handling for extreme values
3. **Test Against sklearn** - Compare IRLS coefficients to sklearn's LogisticRegression
4. **Handle Rare Categories** - Programs with < 20 applications should use hierarchical fallback

### For Product Managers

1. **Expand Geographic Scope** - Consider BC, Alberta, other provinces
2. **User Feedback Loop** - Collect actual admission outcomes to improve model
3. **Fairness Analysis** - Analyze predictions by demographic factors
4. **Supplementary Applications** - Flag programs where prediction is less reliable

### Data Quality Concerns

1. **Self-Report Bias** - Accepted students may report more often
2. **Time Drift** - Admission standards change year-over-year
3. **Missing Data Strategy** - Handle incomplete grade information
4. **101 vs 105 Applicants** - Domestic/international students in different pools
